# Handoff — I-015: Shell redesign slice 2 — register-first landing

**Branch:** `feat/I-015-register-first-landing`

**Dependencies:** I-014 merged

**Context:**
Slice 2 of the shell redesign (`docs/shell-redesign-plan.md`). Currently the shell lands on the "overview" dashboard view. The register is the product — users should arrive directly in the ledger of their last-used account, not a dashboard of empty cards. The dashboard remains accessible via the activity bar overview icon.

**Acceptance criteria:**

1. Default `activeView` state in `App.tsx` changes from `"overview"` to `"ledger"`.

2. A `LAST_ACCOUNT_ID_STORAGE_KEY` constant is added to `apps/web/src/app/use-book-runtime.ts` alongside the existing `LAST_BOOK_ID_STORAGE_KEY`.

3. A `selectLedgerAccount(accountId: string)` helper function is added in `App.tsx` that calls `setSelectedLedgerAccountId(accountId)` and writes the value to `localStorage`. All existing call sites of `setSelectedLedgerAccountId` in `App.tsx` and child components are replaced with `selectLedgerAccount`.

4. After `loadedBook` is resolved (book load effect), the initial account is determined in this order:
   - Read `localStorage` for `LAST_ACCOUNT_ID_STORAGE_KEY`
   - If the stored ID exists in `loadedBook.accounts`, use it as the initial `selectedLedgerAccountId`
   - Otherwise fall back to the first account with `type === "asset"` sorted by `code`
   - If no asset accounts exist, fall back to `loadedBook.accounts[0]`
   - If the book has no accounts, leave `selectedLedgerAccountId` unset

5. On fresh load with no `localStorage` entry: shell opens in ledger view showing the first asset account register.
6. On reload after selecting an account: shell opens in ledger view showing the previously selected account.
7. Overview dashboard is still reachable via the overview icon in the activity bar.
8. `pnpm --filter @tally/web typecheck` passes.
9. `pnpm ci:verify` passes.

**Key files:**

- `apps/web/src/app/App.tsx`
  - Change `useState<BookView>("overview")` → `useState<BookView>("ledger")`
  - Add `selectLedgerAccount` helper that writes to localStorage
  - Replace `setSelectedLedgerAccountId` call sites with `selectLedgerAccount`
  - Add initial account resolution logic in the book-load effect
- `apps/web/src/app/use-book-runtime.ts`
  - Add `LAST_ACCOUNT_ID_STORAGE_KEY = "tally.lastAccountId"` constant (exported)

**Do not touch:**
- Any files outside `apps/web/`
- Any API, domain, or persistence code
- The overview view itself — it stays as-is, just no longer the default
