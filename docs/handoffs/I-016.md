# Handoff — I-016: Shell redesign slice 3 — inspector panel redesign

**Branch:** `feat/I-016-inspector-redesign`

**Dependencies:** I-015 merged

**Context:**
Slice 3 of the shell redesign (`docs/shell-redesign-plan.md`). The inspector panel is currently always visible at 300px, filled with developer notes and placeholder text. It should be collapsed by default, open on demand, and show genuinely useful contextual content: account summary when nothing is selected, transaction detail when a row is selected.

**Acceptance criteria:**

1. **Collapsed by default.** `isInspectorOpen` boolean state added to `App.tsx`, defaulting to `false`.

2. **Grid adapts to collapsed state.** In `styles.css` the shell grid column 4 uses `--inspector-width`:
   - `--inspector-width: 0px` when collapsed; `--inspector-width: 280px` when open
   - `overflow: hidden` on the inspector `<aside>` prevents content bleed when collapsed
   - `transition: width 150ms ease-out` on the inspector `<aside>` for smooth toggle

3. **Toggle mechanisms:**
   - Keyboard: `i` key toggles `isInspectorOpen` (wired in the existing keyboard handler in `shell.ts` or `App.tsx`; suppressed when focus is inside an input, textarea, or select)
   - A toggle button rendered as a thin vertical strip on the left edge of the inspector, visible at all times. Clicking it toggles. It can use a Phosphor `CaretLeft` / `CaretRight` icon (8px, weight `light`).

4. **No-selection state** (inspector open, `ledgerBook.selectedTransaction` is null, ledger view active):
   Shows account summary for the currently selected account:
   ```
   [account name]          [account type]
   ─────────────────────────────────────
   Cleared                  $x,xxx.xx
   Pending                  $x,xxx.xx
   Scheduled (30d)          $x,xxx.xx
   ```
   - **Cleared:** sum of postings to the active account where `cleared === true`
   - **Pending:** sum of all postings to the active account in the current period (cleared + uncleared)
   - **Scheduled 30d:** sum of `postings` amounts for the active account across all `book.scheduledTransactions` where `nextDueOn` is within 30 days of today
   - All amounts rendered in JetBrains Mono, right-aligned, with `--amount-positive` / `--amount-negative` color
   - If no account is selected, show: "Select an account to see its summary." in `--text-muted`

5. **Transaction-selection state** (inspector open, `ledgerBook.selectedTransaction` is not null):
   Four sections rendered in `ShellSidePanels.tsx`:

   **a. Header**
   ```
   [occurredOn]   [description]
   [payee or "—"]
   ```

   **b. Splits** — compact table of all postings on the selected transaction:
   - Account name (left, truncated with ellipsis if needed) + amount (right, JetBrains Mono, positive/negative color)
   - A 1px `--border` separator below the table

   **c. Audit trail** — filter `book.auditEvents` where `entityId === selectedTransaction.id` (the field name may be `entityId` or inside `entityIds` — check the type definition and handle both). Show up to 10 events, newest first:
   - Format: `[HH:mm date]  [human label]  [actor]`
   - Human labels: `transaction.posted` → "Posted", `transaction.updated` → "Updated", `transaction.soft-deleted` → "Deleted", `transaction.restored` → "Restored" — any unrecognised type renders the raw type string
   - If no events match, show "No audit history." in `--text-muted`

   **d. Schedule (conditional)** — only if `selectedTransaction.scheduleId` is set:
   - Look up the schedule in `book.scheduledTransactions` by `id === selectedTransaction.scheduleId`
   - If found, show: schedule `name` + `nextDueOn`
   - Omit this section entirely if no `scheduleId`

   **e. Attachments placeholder** — one line: "Attachments · coming soon" in `--text-muted`. Always shown in this state.

6. **Non-ledger views:** when `activeView !== "ledger"`, the inspector toggle button remains, but the content area is empty (no inspector content for non-ledger views — remove the existing per-view placeholder text from `ShellSidePanels.tsx` entirely).

7. `pnpm --filter @tally/web typecheck` passes.
8. `pnpm ci:verify` passes.
