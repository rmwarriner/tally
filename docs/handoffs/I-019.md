## Changes required

### 1. Apply `--cell-pad-y` to table cells

Currently `.register-table td` has only `border-bottom` — no padding. Add explicit vertical and horizontal padding so row height comes from intentional spacing, not browser defaults.

In `styles.css`, update the existing `.register-table th, .register-table td` rule:

```css
.register-table th,
.register-table td {
  padding: var(--cell-pad-y) 8px;
  vertical-align: middle;
  border-bottom: 1px solid color-mix(in srgb, var(--border) 35%, transparent);
}
```

Header cells need tighter top padding — add a separate rule:

```css
.register-table th {
  padding-top: 6px;
  padding-bottom: 6px;
}
```

---

### 2. Style the register table headers

Currently `th` inherits only the border-bottom. Add:

```css
.register-table th {
  font-family: var(--mono-font);
  font-size: var(--text-xs);
  font-weight: var(--weight-semibold);
  color: var(--text-muted);
  text-transform: uppercase;
  letter-spacing: 0.06em;
  white-space: nowrap;
}
```

---

### 3. Description column — medium weight

The description is the primary identifier in a register row. Give it `font-weight: var(--weight-medium)` to read as slightly more prominent than the muted payee alongside it.

In the existing `.register-col-description, .register-cell-description` rule, add:

```css
font-weight: var(--weight-medium);
```

---

### 4. Inline edit inputs — embedded appearance

The global `input` rule gives all inputs `background: var(--surface-input); border: 1px solid var(--border); border-radius: 10px`. Inside a register row during inline editing, these look like form elements dropped into a table. They need to look embedded in the row.

Add a scoped rule that strips the default input appearance for inputs inside `.register-row`:

```css
.register-row input:not([type="checkbox"]) {
  background: transparent;
  border: none;
  border-radius: 0;
  border-bottom: 1px solid color-mix(in srgb, var(--accent) 55%, transparent);
  padding: 2px 4px;
  width: 100%;
  font-size: var(--text-sm);
}

.register-row input:not([type="checkbox"]):focus {
  outline: none;
  border-bottom-color: var(--accent);
}
```

This applies to the date, description, and payee inline edit inputs as well as the new-row draft inputs at the top of the register.

---

### 5. Saving row state

When a save is in flight (`props.busy !== null` and the row is the one being edited), the row needs a visible but non-disruptive indicator.

**In `styles.css`** — add after the existing `.register-row.editing td` block:

```css
.register-row.saving td {
  background: var(--accent-soft);
  opacity: 0.55;
  pointer-events: none;
}
```

**In `LedgerRegisterPanel.tsx`** — update the `className` computation for each transaction `<tr>` to include `saving` when appropriate:

```tsx
className={[
  "register-row",
  props.selectedLedgerTransactionId === transaction.id ? "selected" : "",
  isEditingRow ? "editing" : "",
  isEditingRow && props.busy !== null ? "saving" : "",
].filter(Boolean).join(" ")}
```

**In `LedgerRegisterPanel.tsx`** — update the inline edit Save button label from the static `"Save"` to reflect the saving state:

```tsx
{isEditingRow && props.busy !== null ? "Saving…" : "Save"}
```
