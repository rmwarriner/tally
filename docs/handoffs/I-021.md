## Context

Activity bar icon backgrounds extend downward excessively, making buttons look droopy. The cause is in `styles.css`:

```css
/* current */
.shell-activity-button,
.shell-activity-settings {
  border: 0;
  border-radius: 10px;
  background: color-mix(in srgb, var(--activity-bar-text) 8%, transparent);
  color: inherit;
  padding: 8px 0;   /* ← vertical-only padding, no horizontal */
}
```

Buttons fill 100% of `.shell-activity-nav` width (the nav has `width: 100%` and uses `display: grid`). With `padding: 8px 0`, a 20px icon renders a button that is ~36px tall but as wide as the activity bar — a horizontal rectangle. With `border-radius: 10px`, the corners bite more into the top and bottom than the sides, producing the droopy appearance.

The fix is to make each button a centered square: explicit equal dimensions, centered within the nav column, rather than stretching to fill the full nav width.

---

## Changes required

All changes are in `apps/web/src/app/styles.css` only. No changes to `ShellActivityBar.tsx` are needed.

### 1. Square up the button shape

Replace the existing `.shell-activity-button, .shell-activity-settings` rule:

```css
.shell-activity-button,
.shell-activity-settings {
  border: 0;
  border-radius: 10px;
  background: color-mix(in srgb, var(--activity-bar-text) 8%, transparent);
  color: inherit;
  padding: 0;
  width: 36px;
  height: 36px;
  display: flex;
  align-items: center;
  justify-content: center;
  margin: 0 auto;
}
```

The 36px dimension gives 8px of visual breathing room around the 20px icon on each side, matching the previous vertical padding while now being consistently applied horizontally too.

### 2. Remove the width override on the settings button

The existing `.shell-activity-settings` rule adds `width: 100%` which would override the square sizing above. Update it to remove that property:

```css
.shell-activity-settings {
  margin-top: auto;
}
```
