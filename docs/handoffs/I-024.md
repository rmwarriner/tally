## Context

The search bar above the register duplicates the command palette / spotlight entry in the upper right. It is being removed. A per-tab floating finder may return in a future issue — do not build a replacement, just remove.

---

## Changes required

### 1. `LedgerRegisterPanel.tsx` — remove the search input block

Remove the entire `<label className="ledger-filter">` block that contains the search input (the first child of `.ledger-toolbar`):

```tsx
// REMOVE this block entirely:
<label className="ledger-filter">
  <span className="muted">Search register</span>
  <input
    ref={props.ledgerSearchInputRef}
    data-testid="ledger-search-input"
    value={props.ledgerSearchText}
    placeholder="description, payee, account, tag, status"
    onChange={(event) => props.setLedgerSearchText(event.target.value)}
  />
</label>
```

Also remove the hotkeys hint paragraph immediately after the chip rows:

```tsx
// REMOVE this block entirely:
<p className="form-hint">
  Hotkeys: `/` search, `j` or down move later, `k` or up move earlier, `Esc` clear
  selection, `Ctrl/Cmd+Shift+[ ]` switch tabs. Search supports tags, account code/name, and
  status tokens.
</p>
```

### 2. `LedgerRegisterPanel.tsx` — remove props

Remove from `LedgerRegisterPanelProps`:

```tsx
ledgerSearchInputRef: RefObject<HTMLInputElement | null>;
ledgerSearchText: string;
setLedgerSearchText: Dispatch<SetStateAction<string>>;
```

Remove the `RefObject` import from `react` if it is now unused (check — it may still be used by other refs in the component).

### 3. `LedgerMainPanels.tsx` — remove props and pass-throughs

Remove from `LedgerMainPanelsProps`:

```tsx
ledgerSearchInputRef: RefObject<HTMLInputElement | null>;
ledgerSearchText: string;
setLedgerSearchText: Dispatch<SetStateAction<string>>;
```

Remove the corresponding three prop pass-throughs to `<LedgerRegisterPanel>`:

```tsx
ledgerSearchInputRef={props.ledgerSearchInputRef}
ledgerSearchText={props.ledgerSearchText}
setLedgerSearchText={props.setLedgerSearchText}
```

Remove the `RefObject` import from `react` in this file if now unused.

### 4. `App.tsx` — remove state, ref, and wiring

**Remove the ref declaration:**
```tsx
const ledgerSearchInputRef = useRef<HTMLInputElement | null>(null);
```

**Remove `ledgerSearchText` from per-tab state.** Each tab entry currently initialises with `ledgerSearchText: ""` — remove that field from the tab state object type and its initialiser.

**Remove the derived value:**
```tsx
const ledgerSearchText = activeLedgerRegisterTab?.ledgerSearchText ?? "";
```

**Remove `searchText` from the `createLedgerBookModel` call** — find the call that passes `searchText: ledgerSearchText` and remove that argument.

**Remove the `setLedgerSearchText` function** (the one that updates `tab.ledgerSearchText` inside the tab array).

**Remove the keyboard shortcut** that focuses the search input. Look for `ledgerSearchInputRef.current?.focus()` and remove the entire shortcut handler for `/` (or the single line if it is inline within a larger handler).

**Remove the three prop bindings** passed to `<LedgerMainPanels>`:
```tsx
ledgerSearchInputRef={ledgerSearchInputRef}
ledgerSearchText={ledgerSearchText}
setLedgerSearchText={setLedgerSearchText}
```

### 5. `shell.ts` — remove `searchText` parameter from `createLedgerBookModel`

Find `createLedgerBookModel` and remove the `searchText` parameter from its input and any filtering logic that uses it. If the function signature is typed separately, update the type too.
