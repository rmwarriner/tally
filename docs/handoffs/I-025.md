## Context

The tab content area currently renders three sections below the register table: an Advanced Editor panel, a Ledger Operations / Reconciliation panel, and a Balances panel. All three are being removed. The register is the entire content of a tab.

The Advanced Editor (transaction split editor) still needs to be accessible via the "Advanced" item in the row ellipsis menu — retain the open/close state and the rendered panel, but remove the placeholder "Optional" card that appears when it is closed. The actual editor appears only when active, so it contributes no persistent height in default state after this change.

The Ledger Operations panel (reconciliation) and the Balances panel are removed entirely with no replacement.

### 2. `LedgerMainPanels.tsx` — remove Advanced Editor placeholder card

The current logic renders either the full editor or an "Optional" placeholder card:

```tsx

  <>
    <div className="posting-editor-row">
      <button type="button" onClick={props.onToggleLedgerDetailOpen}>
        Hide advanced editor
      </button>
    </div>
    {props.transactionEditorPanel}
  </>
) : (
  <article className="panel">
    <div className="panel-header">
      <span>Advanced editor</span>
      <span className="muted">Optional</span>
    </div>
    <p className="form-hint">
      Inline row editing is the default. Use the row-level Advanced action only when you need
      split-level editing details.
    </p>
  </article>
)}
```

Replace with — render the editor pane only when open, nothing when closed:

```tsx
{props.isLedgerDetailOpen ? (
  <>
    <div className="posting-editor-row">
      <button type="button" onClick={props.onToggleLedgerDetailOpen}>
        Hide advanced editor
      </button>
    </div>
    {props.transactionEditorPanel}
  </>
) : null}
```
