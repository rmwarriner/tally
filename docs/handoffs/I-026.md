## Context

The register toolbar has a row of four chip-style buttons for status filtering: "All statuses", "open", "cleared", "reconciled". These are visually heavy and take up a full toolbar row. Replace them with a single compact `<select>` element offering the same four options. Filter behavior is unchanged.

---

## Changes required

### 1. `LedgerRegisterPanel.tsx` — replace the status chip row

Find the `<div className="ledger-chip-row">` that renders the four status buttons:

```tsx
// REMOVE:
<div className="ledger-chip-row">
  {(["all", "open", "cleared", "reconciled"] as const).map((status) => (
    <button
      key={status}
      className={`ledger-chip${props.ledgerStatusFilter === status ? " active" : ""}`}
      type="button"
      onClick={() => props.setLedgerStatusFilter(status)}
    >
      {status === "all" ? "All statuses" : status}
    </button>
  ))}
</div>
```

Replace with:

```tsx
<select
  className="ledger-status-select"
  value={props.ledgerStatusFilter}
  onChange={(event) =>
    props.setLedgerStatusFilter(
      event.target.value as "all" | "cleared" | "open" | "reconciled",
    )
  }
>
  <option value="all">All statuses</option>
  <option value="open">Uncleared</option>
  <option value="cleared">Cleared</option>
  <option value="reconciled">Reconciled</option>
</select>
```

Note: option labels for "open" and "cleared" match the inline edit status select introduced in I-022 ("Uncleared" / "Cleared" / "Reconciled") for consistency.
