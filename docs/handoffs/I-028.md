## Context

Currently, clicking an account in the COA panel calls `setSelectedLedgerAccountId` — which just filters the existing register without changing the tab or navigating. The new behavior:

- **Single-click** → load that account's ledger in the currently active tab (replace whatever account it was showing)
- **Right-click** → show a small context menu with "Open in new tab"; selecting it opens a new tab for the account and makes it active

The `onAccountSelect` prop on `CoaSidebar` currently handles the old filtering behavior. We will repurpose it as "load in active tab" and add `onOpenInNewTab` for right-click.

Replace `onAccountSelect` with two dedicated callbacks and add a context menu state:

### 2. `CoaSidebar.tsx` — context menu state

Add local state for the open context menu:

```tsx
const [contextMenu, setContextMenu] = useState<{
  accountId: string;
  x: number;
  y: number;
} | null>(null);
```

Add a `useEffect` to close the context menu on outside click:

```tsx
useEffect(() => {
  if (!contextMenu) return;
  function handleClick() {
    setContextMenu(null);
  }
  document.addEventListener("mousedown", handleClick);
  return () => document.removeEventListener("mousedown", handleClick);
}, [contextMenu]);
```

Add the `useState` and `useEffect` imports from React (already has `useState` from I-029 — add `useEffect`).

At the bottom of the `<section>` return, before the closing tag, add:

```tsx

  <div
    className="coa-context-menu"
    style={{ position: "fixed", top: contextMenu.y, left: contextMenu.x }}
```

### 5. `App.tsx` — add `loadAccountInActiveTab` function

This replaces the active tab's account without creating a new tab:

```tsx
function loadAccountInActiveTab(accountId: string) {
  setLedgerRegisterTabs((currentTabs) =>
    currentTabs.map((tab) =>
      tab.id === activeLedgerRegisterTabId
        ? { ...tab, selectedLedgerAccountId: accountId, selectedLedgerTransactionId: null }
        : tab,
    ),
  );
  setSelectedLedgerAccountId(accountId);
}
```
